class giraffe.graph_data_builder.UniGraphDataBuilder(source_table, node_entity, link_entities, ts_col, schema, prefix, okta_username, entity_id_map_tb_name=None, hotkey_th=None, verbose=True, cluster='vega')[source]

Bases: giraffe.graph_data_builder.BaseGraphDataBuilder

Class to build and retrieve graph data whose node is a single type of entity and whose relation is defined by link_entities.

For instance, suppose that we aim to build a graph where nodes correspond to users, and a pair of users is connected if they share the same ip, or device. In this case, we need provide as input a source table in which each row contains <user_id, ip, device_id, event_time> (event_time indicates the time when a user used the corresponding device or ip, for example). Then, we set node_entity to be user_id, link_entities to be ['ip', 'device_id'], instantiate the class, and call its methods to obtain the desired graph.

Parameters

  • source_table (str) – name of souce table in database; it must contain columns specified in node_entity, link_entities, ts_col.

  • node_entity (str) – name of entity considered as node

  • link_entities (list) – list of entities. A pair of node_entity is connected if they share at least one entity in link_entities

  • ts_col (str) – column name of timestamp when the link is effective; should have format yyyy-MM-dd hh:mm:ss

  • schema (str) – schema name, for instance, sandbox_analytics_au

  • prefix (str) – prefix of the resulting table, for intance, test_

  • okta_username (str) – okta username

  • entity_id_map_tb_name (str) – the name of the table that maps entity raw id to integer id; defaults to None; if left default, the table is named '{schema}.{prefix}entity_id_map'

  • hotkey_th (int) – defaults to None, no filtering hotkey if set to None; otherwise, filter hotkey if occurrence greater than hotkey_th

  • verbose (bool) – True to print details, defaults to True

  • cluster (str) – defaults to vega

class giraffe.graph_data_builder.BiGraphDataBuilder(source_table, left_entity, right_entity, ts_col, schema, prefix, okta_username, entity_id_map_tb_name=None, verbose=True, cluster='vega')[source]

Bases: giraffe.graph_data_builder.BaseGraphDataBuilder

Class to build and retrieve bipartite graph data whose nodes are composed of two kinds of entities.

For instance, suppose that we aim to build a graph where nodes correspond to users and used device, and a user and a device are connected if the user has used the device. In this case, we need provide as input a source table in which each row contains <user_id, device_id, event_time> (event_time indicates the time when a user used the corresponding device, for example). Then, we set left_entity to be user_id, right_entity to be device_id, instantiate the class, and call its methods to obtain the desired graph.

Parameters

  • source_table (str) – name of souce table in database; it must contain columns specified in left_entity, right_entity, ts_col.

  • left_entity (str) – name of entity considered as left node

  • right_entity (str) – name of entity considered as right node

  • ts_col (str) – column name of timestamp when the link is effective; should have format yyyy-MM-dd hh:mm:ss

  • schema (str) – schema name, for instance, sandbox_analytics_au

  • prefix (str) – prefix of the resulting table, for intance, test_

  • okta_username (str) – okta username

  • entity_id_map_tb_name (str) – the name of the table that maps entity raw id to integer id; defaults to None; if left default, the table is named '{schema}.{prefix}entity_id_map'

  • verbose (bool) – True to print details, defaults to True

  • cluster (str) – defaults to vega
